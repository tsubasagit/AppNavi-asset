# ワークフロー競合の修正

## 問題

複数のGitHub Pagesワークフローが存在し、競合していました：

1. **deploy-pages.yml** - `dist` ディレクトリをデプロイ（APIファイルを含む）✅
2. **deploy-simple.yml** - `dist` ディレクトリをデプロイ（APIファイルを含まない）❌
3. **deploy-templates.yml** - `_site` ディレクトリをデプロイ（APIファイルを含む）

### 原因

`deploy-simple.yml` と `deploy-pages.yml` は同じ `concurrency.group: "pages"` を使用しているため、どちらかが実行されると、もう一方がキャンセルされる可能性があります。

また、`deploy-simple.yml` にはAPIファイルをコピーする処理がないため、これが実行されると、APIファイルがデプロイされず、404エラーが発生します。

## 修正内容

`deploy-simple.yml` に以下を追加しました：

1. APIファイルのコピー処理
2. `.nojekyll` ファイルの作成

これにより、どちらのワークフローが実行されても、APIファイルが正しくデプロイされます。

## 推奨事項

将来的には、以下のいずれかの対応を推奨します：

1. **`deploy-simple.yml` を削除する**（推奨）
   - `deploy-pages.yml` のみを使用する

2. **`deploy-simple.yml` を無効化する**
   - ワークフローファイルを削除するか、`on:` セクションをコメントアウト

3. **両方のワークフローを統合する**
   - 1つのワークフローファイルに統合する
